Typed Multilayer Agent Runtime

A possible enterprise-grade direction: each layer has its own graph semantics, contracts, observability, and safety controls.

Proposed Product Shape
Layer 4
Agent State Machines: plan -> call model/tool -> observe -> decide -> loop -> finish/escalate
Layer 3
Typed AI Dataflow: query, embedding, retrieval, documents, prompt, generation, validation, evaluation
Layer 2
Durable Execution Graph: checkpoints, retries, leases, artifacts, deadlines, compensation, human gates
Layer 1
Enterprise Control Plane: projects, RBAC, audit, secrets, policy, budgets, deployment targets, observability
A

Enterprise-first foundation

Build typed agents around governance, auditability, policy, budgets, replay, and deployment controls from the start.

B

Agent-builder-first foundation

Build the authoring model and runtime loop first, then add governance features once the agent graph feels right.
